# Requirement Checklist

Evaluate prospect artifacts (context.md + requirements.md) against this checklist.

## Categories

### 1. Completeness

| ID      | Criterion                                         | Status |
|---------|----------------------------------------------------|--------|
| CHK-101 | Every FR has at least one acceptance criterion      |        |
| CHK-102 | Every US has priority assigned (P1/P2/P3)           |        |
| CHK-103 | Problem Statement exists and is non-trivial         |        |
| CHK-104 | Technical Decisions section with at least one TD    |        |
| CHK-105 | Key Entities / Domain Model section exists          |        |

### 2. Measurability

| ID      | Criterion                                         | Status |
|---------|----------------------------------------------------|--------|
| CHK-201 | No FR uses vague adjectives without metrics         |        |
| CHK-202 | NFRs have numeric thresholds                        |        |
| CHK-203 | Success Criteria are quantifiable or binary          |        |

### 3. Testability

| ID      | Criterion                                         | Status |
|---------|----------------------------------------------------|--------|
| CHK-301 | Each SC maps to a verifiable condition              |        |
| CHK-302 | Acceptance criteria use structured format            |        |
| CHK-303 | No SC relies solely on subjective judgment           |        |

### 4. Scope Clarity

| ID      | Criterion                                         | Status |
|---------|----------------------------------------------------|--------|
| CHK-401 | In Scope lists concrete deliverables                |        |
| CHK-402 | Out of Scope exists with exclusions and rationale    |        |
| CHK-403 | No FR contradicts an Out of Scope item              |        |

### 5. Edge Cases

| ID      | Criterion                                         | Status |
|---------|----------------------------------------------------|--------|
| CHK-501 | At least one error/failure scenario documented       |        |
| CHK-502 | Boundary conditions mentioned for numeric inputs     |        |
| CHK-503 | Concurrency/race conditions addressed (if applicable)|        |
